---
title: Search and paginate the catalog
description: Find a card in Binder's catalog, filter by game, and walk every page with cursor pagination in bash or JavaScript.
---

The catalog is every distinct card any Binder user tracks, in one normalized
cross-game model. This guide finds a card, then walks the entire catalog page by
page. The endpoint is `GET /api/v1/cards` on `https://v0-pokebinder.vercel.app`, no auth.

## Find a card

Pass `q` for a case-insensitive substring match on the name, and optionally
`game` to restrict to `mtg`, `pokemon`, or `onepiece`:

```bash
curl "https://v0-pokebinder.vercel.app/api/v1/cards?q=charizard&game=pokemon"
```

```json
{
  "count": 1,
  "has_more": false,
  "next_cursor": null,
  "cards": [
    {
      "card_id": "base1-4",
      "name": "Charizard",
      "game": "pokemon",
      "set_name": "Base Set",
      "set_code": "base1",
      "collector_number": "4",
      "rarity": "Rare Holo",
      "image_url": "https://v0-pokebinder.vercel.app/api/cards/image?id=base1-4",
      "price": { "usd": 300.0, "usd_foil": null, "eur": null }
    }
  ],
  "attribution": "…"
}
```

Both `q` and `game` are optional. Omit them and you get the whole catalog, one
page at a time.

## Understand the page shape

Every response is one page:

- `count` — the number of cards **in this page**, not the catalog total.
- `has_more` — whether another page exists.
- `next_cursor` — an opaque token for the next page, or `null` when `has_more` is
  `false`.
- `limit` (query param) — page size, 1–100, default 50.

To get the next page, pass `next_cursor` back as `?cursor=`. Cursors are opaque:
don't parse or build them, just hand them back verbatim.

## Paginate with cursors (bash)

This loop walks the entire MTG catalog in pages of 50, printing each card's name:

```bash
cursor=""
while :; do
  url="https://v0-pokebinder.vercel.app/api/v1/cards?game=mtg&limit=50"
  [ -n "$cursor" ] && url="$url&cursor=$cursor"
  resp=$(curl -s "$url")
  echo "$resp" | jq -r '.cards[].name'
  cursor=$(echo "$resp" | jq -r '.next_cursor // empty')
  [ -z "$cursor" ] && break
done
```

Each iteration fetches a page, prints the names, and reads `next_cursor`. When
`next_cursor` is `null`, `jq`'s `// empty` yields an empty string and the loop
stops.

## Paginate with cursors (JavaScript)

The same loop with `fetch`, collecting every card into one array:

```js
async function fetchAllCards({ game, q } = {}) {
  const all = [];
  let cursor = null;

  do {
    const url = new URL("https://v0-pokebinder.vercel.app/api/v1/cards");
    if (game) url.searchParams.set("game", game);
    if (q) url.searchParams.set("q", q);
    url.searchParams.set("limit", "100");
    if (cursor) url.searchParams.set("cursor", cursor);

    const res = await fetch(url);
    if (!res.ok) {
      const { error } = await res.json();
      throw new Error(`${error.code}: ${error.message}`);
    }
    const page = await res.json();
    all.push(...page.cards);
    cursor = page.has_more ? page.next_cursor : null;
  } while (cursor);

  return all;
}

const cards = await fetchAllCards({ game: "mtg" });
console.log(`Fetched ${cards.length} cards`);
```

Use `limit=100` to make the fewest requests, and remember the [60 requests per
minute](/reference/conventions#rate-limits) cap — a very large catalog is well
within it, but a tight retry loop is not.

## Handle errors

An unknown `game` or a tampered `cursor` returns `400` with the
[standard envelope](/reference/conventions#errors):

```json
{ "error": { "code": "invalid_request", "message": "Unknown game; expected mtg, pokemon, or onepiece" } }
```

The JavaScript loop above throws on any non-2xx so a bad parameter surfaces
immediately instead of silently returning nothing.
