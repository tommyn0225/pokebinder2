---
title: Quickstart
description: Fetch a public binder, its history, and a trade list.
---

Every endpoint is a plain `GET` that returns JSON. No headers or keys are needed.

## 1. Make something public

The API only sees public resources:

- **A binder** — open it in the app and choose **Make public**. Its share link
  looks like `/(b)/<binder-id>`; the `<binder-id>` is what you pass below.
- **A trade list** — open **View trades**, choose **Make public**, and copy the
  share link `/(t)/<token>`; the `<token>` is what you pass below.

## 2. Get a public binder

```bash
curl http://localhost:3000/api/v1/binders/<binder-id>
```

```json
{
  "binder": { "id": "…", "name": "My MTG binder", "game": "mtg" },
  "card_count": 12,
  "total_usd": 84.5,
  "holdings": [
    {
      "card_id": "…",
      "name": "Lightning Bolt",
      "game": "mtg",
      "set_name": "Magic 2011",
      "set_code": "m11",
      "collector_number": "149",
      "rarity": "common",
      "image_url": "https://…",
      "quantity": 2,
      "finish": "nonfoil",
      "for_trade": true,
      "price": { "usd": 1.25, "usd_foil": 6.0, "eur": 1.1 }
    }
  ],
  "attribution": "Card data and pricing courtesy of Scryfall (MTG), PokéWallet (Pokémon), and OPTCG API (One Piece)…"
}
```

A missing or private binder returns `404` with the standard error envelope:

```json
{ "error": { "code": "not_found", "message": "Binder not found or not public" } }
```

## 3. Get a binder's value history

Binder's own daily snapshots — not upstream history.

```bash
curl http://localhost:3000/api/v1/binders/<binder-id>/history
```

```json
{
  "binder_id": "…",
  "history": [
    { "day": "2026-07-01T00:00:00+00:00", "total_usd": 80.0 },
    { "day": "2026-07-02T00:00:00+00:00", "total_usd": 84.5 }
  ],
  "attribution": "…"
}
```

## 4. Get a public trade list

```bash
curl http://localhost:3000/api/v1/trades/<token>
```

```json
{
  "total_cards": 5,
  "cards": [
    {
      "card_id": "…",
      "name": "Charizard",
      "game": "pokemon",
      "set_name": "Base Set",
      "quantity": 1,
      "finish": "nonfoil",
      "price": { "usd": 300.0, "usd_foil": null, "eur": null }
    }
  ],
  "attribution": "…"
}
```

## 5. Search the card catalog

The catalog needs no IDs at all — it's every card tracked in Binder, in one
normalized model across all three games:

```bash
curl "http://localhost:3000/api/v1/cards?q=charizard&game=pokemon"
```

```json
{
  "count": 1,
  "has_more": false,
  "next_cursor": null,
  "cards": [
    {
      "card_id": "base1-4",
      "name": "Charizard",
      "game": "pokemon",
      "set_name": "Base Set",
      "set_code": "base1",
      "collector_number": "4",
      "rarity": "Rare Holo",
      "image_url": "https://…",
      "price": { "usd": 300.0, "usd_foil": null, "eur": null }
    }
  ],
  "attribution": "…"
}
```

`game` and `q` are optional; `limit` sets the page size (default 50, max 100).
`count` is the number of cards in **this page**, not the catalog total. When
`has_more` is true, pass `next_cursor` back as `?cursor=…` to fetch the next
page:

```bash
curl "http://localhost:3000/api/v1/cards?q=charizard&cursor=<next_cursor>"
```

## 6. Get one card with its price history

Use a `card_id` from the search above. The history is Binder's own daily
snapshots, one point per day:

```bash
curl http://localhost:3000/api/v1/cards/pokemon/base1-4
```

```json
{
  "card": { "card_id": "base1-4", "name": "Charizard", "game": "pokemon", "…": "…" },
  "history": [
    { "day": "2026-07-01", "price_usd": 295.0 },
    { "day": "2026-07-02", "price_usd": 300.0 }
  ],
  "attribution": "…"
}
```

A card no Binder user tracks returns `404`:

```json
{ "error": { "code": "not_found", "message": "Card not found in catalog" } }
```

## 7. Get a card's price history over a window

Binder's own daily snapshots for one card, bounded by `days` (default 30, max
365) — one point per day:

```bash
curl "http://localhost:3000/api/v1/cards/pokemon/base1-4/history?days=7"
```

```json
{
  "game": "pokemon",
  "card_id": "base1-4",
  "days": 7,
  "history": [
    { "day": "2026-07-07", "price_usd": 298.0 },
    { "day": "2026-07-08", "price_usd": 300.0 }
  ],
  "attribution": "…"
}
```

## Notes

- Every error uses the envelope `{ "error": { "code", "message" } }`. Codes:
  `invalid_request`, `not_found`, `rate_limited`, `internal`.
- Requests are rate-limited per IP (60/minute). Over the limit returns `429`
  with a `Retry-After` header.
- Responses are cached for 5 minutes (`Cache-Control: public, max-age=300`) and
  carry a weak `ETag`. Send it back as `If-None-Match` to get a `304 Not
  Modified` when nothing changed.
- CORS is open (`Access-Control-Allow-Origin: *`) so you can call the API from a
  browser app.
- The machine-readable spec is at `/api/v1/openapi.json`.
