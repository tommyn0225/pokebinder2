The **Binder Public API** exposes Binder's own value-added layer — public binders,
their valuation and price history, public trade lists, and Binder's normalized
card catalog. It is read-only and requires no authentication.

## What it is (and isn't)

Binder is built on three free upstream sources: **Scryfall** (Magic: The
Gathering), **PokéWallet** (Pokémon), and **OPTCG API** (One Piece). Their terms
allow value-added apps, but not simply proxying or republishing their data.

So this API serves **our layer** — binders, holdings, valuation, the price
history we build ourselves from daily snapshots, and our normalized card
catalog — and deliberately does **not** offer a live upstream card or price
search passthrough. Every response includes an `attribution` field crediting
the upstream sources.

## Building on the catalog

Want to build your own collection app? The **card catalog** is the piece meant
for you. It covers every card tracked in Binder, in one normalized model across
all three games — same fields whether a card is MTG, Pokémon, or One Piece —
plus a per-card daily USD price history that Binder builds itself. A card enters
the catalog when any Binder user tracks it, so the catalog grows with the app;
it is not an exhaustive index of every card ever printed. For that, go to the
upstream sources directly ([Scryfall](https://scryfall.com/docs/api),
[PokéWallet](https://www.pokewallet.io/api-docs),
[OPTCG API](https://optcgapi.com/documentation)) — they're free, and it keeps
everyone inside their terms of use.

## Authentication

None. There are no API keys.

The API only returns resources whose owner has explicitly made them **public**
(a public binder, or a trade list toggled to public). Anything private — or that
doesn't exist — returns `404`.

## Base URL

During local development:

```
http://localhost:3000
```

<Note>
  A production base URL will be added once the app is deployed. Until then, run
  the Binder app locally and call the API at `localhost:3000`.
</Note>

## Endpoints

| Method | Path | Description |
| ------ | ---- | ----------- |
| `GET` | `/api/v1/binders/{id}` | A public binder with holdings and total value |
| `GET` | `/api/v1/binders/{id}/history` | A public binder's daily value history |
| `GET` | `/api/v1/trades/{token}` | A public trade list |
| `GET` | `/api/v1/cards` | Search the normalized card catalog |
| `GET` | `/api/v1/cards/{game}/{id}` | A catalog card with our daily price history |
| `GET` | `/api/v1/openapi.json` | This OpenAPI document |

See the [Quickstart](/quickstart) for end-to-end examples, or the
**API reference** tab for the full schema.
